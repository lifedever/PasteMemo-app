1. 架构总览

┌──────────────────────────────────────────────────────────────────┐ │ PasteMemo macOS Client │ │ ClipboardManager ──► ClipItem (SwiftData, KSUID itemID) │ │ │ SyncOrigin.stampLocal ▲ importPayload │ │ ▼ │ (origin_* 三元组补全) │ │ SyncScheduler (@MainActor) ──► SyncEngine.performSync() │ │ │ reschedule/stop ├─ prepareUploadPayloads (PMEM) │ │ │ (autoPaused, lastErr)├─ uploadBatchAdaptive → POST /sync│ │ │ ├─ pullFromPeer → GET /pull │ │ │ └─ syncTrashFromClient → GET /trash │ │ SyncSettingsView / RevealableSecureField / SyncEncryption │ │ KSUID.generate / SyncCrypto (PBKDF2+AES-GCM) │ └───────────────────────────────┬──────────────────────────────────┘ │ HTTPS (Bearer on write APIs) ▼ ┌──────────────────────────────────────────────────────────────────┐ │ sync-server (Go 1.22+) │ │ httpserver ─► store (SQLite + item_attachments) │ │ │ gzip + Bearer 中间件；clientIP(trust_proxy) │ │ │ cron PurgeExpiredTrash (TrashRetentionDays=10) │ │ web/index.html (//go:embed) ── Web Crypto + sips(HEIC) 转码 │ │ imagetype (DetectMIME/AsJPEG) + pmemcrypto (验证) │ └──────────────────────────────────────────────────────────────────┘

同步为中心化存储 + 多客户端增量交换模式：各 Client 以自身 client_id（27 字符 base62 KSUID）为命名空间上传；拉取时读取对端命名空间；item_id 同样为 27 字符 base62 KSUID 跨设备主键。无 P2P、无 CRDT。

4. 同步引擎

入口：SyncEngine.performSync(container:progress:)（@MainActor 枚举，内部 async HTTP）

4.1 流程顺序

1. 读取配置 (URL, token, clientID, since=lastCompletedAt) 2. fetchItems(createdAt > since) → filter isLocalOrigin 3. fetchRemoteClients (GET /clients) → peerByID 字典 4. FOR each batch (size=syncBatchSize): prepareUploadPayloads → uploadBatchAdaptive (POST /sync) 自适应: body > 48 MiB 则二分 batch；单条超限则 stripBinaryFields 标 truncated 5. FOR each selected peer: pullFromPeer (GET /pull paginated) 6. FOR self + peers: syncTrashFromClient (GET /trash paginated) 7. uploaded==0 && downloaded==0 && deleted==0 ⇒ throw .noItems 8. 返回 SyncResult(uploaded, downloaded, deleted)

4.2 上传实现

buildPayload(from:localClientID:) — 序列化 ClipItem；补全 origin 三元组；Base64 编码 blob。
uploadBatch — 构造 SyncUploadRequest；若启用加密则 prepareUploadPayloads 将完整 inner JSON 加密为 PMEM Base64。
uploadBatchAdaptive — 请求体 > 48 MiB 时二分拆分 batch；单条仍过大则 stripBinaryFields 后再试。

ISO8601：ISO8601DateFormatter，.withInternetDateTime, .withFractionalSeconds。

4.3 拉取实现

pullFromPeer 查询参数：

参数来源
client_id对端 peerID（必填）
sinceSyncSettings.lastPullAt(peerID)（ISO8601）
limitsyncBatchSize（默认 20，服务端 clamp 到 1–100）
cursor_created_at下一页起点 created_at
cursor_item_id下一页起点 item_id

解密：decryptPayloadIfNeeded → JSON decode 为 SyncClipPayload。

导入：importPayload — dedup by itemID；origin 三元组优先取 payload 字段，缺失时回退到对端元数据（peerID / peer?.lastHostname / peer?.lastIP）。

全部页成功后：SyncSettings.setLastPullAt(peerID, Date())。

4.4 回收站同步

目标列表：syncTrashTargets = 本机 clientID + 所有 selected peers（去重，seen.insert(key)）。

GET /api/v1/trash，client_id 必填，since = lastTrashSyncAt(clientID)，分页用 cursor_deleted_at + cursor_item_id。

本地：deleteLocalItems(itemIDs:) 用 FetchDescriptor<ClipItem> 按 itemID 查找并调 ClipItemStore.deleteAndNotify。

服务端软删除由 Web UI DELETE /api/v1/items/{client}/{item} 触发；保留 10 天（store.TrashRetentionDays = 10）后由定时任务 PurgeExpiredTrash 永久删除。

4.5 本机条目过滤

文件：Sources/Engine/Sync/SyncOrigin.swift

static func isLocalOrigin(originClientID: String?, localClientID: String) -> Bool {
    guard let originClientID, !originClientID.isEmpty else { return true }  // legacy
    return SyncSettings.clientIDsMatch(originClientID, localClientID)
}

在 performSync 中：localItems.filter { isLocalOrigin(...) }，全量重同步（since == nil）同样生效。

5. 调度与配置

5.1 SyncScheduler

  • @MainActor @Observable；SyncScheduler.shared 单例。
  • start(container:) — AppDelegate.applicationDidFinishLaunching 调用；reschedule() 决定是立即触发 syncNow() 还是安排 Timer。
  • stop() — AppDelegate.applicationWillTerminate 调用；也用于 autoPaused 暂停时。
  • resumeAutoSync() — 清 autoPaused、重新 reschedule()。
  • syncNow() 防重入 isSyncing；成功路径写 lastCompletedAt、lastSyncCount/Downloaded/Deleted、清 lastError、autoPaused=false、reschedule()。
  • 失败路径：写 lastError、autoPaused=true、stop()（暂停定时器）。
  • SyncError.noItems 视为成功（仍推进 lastCompletedAt 并清错误），避免无限重试空客户端。
  • nextSyncDate = lastCompletedAt + interval，仅在 enabled && !autoPaused 时返回非 nil。

6. 加密实现

6.1 算法参数（Swift / Go / Web 一致）

步骤参数
PBKDF2HMAC-SHA256，600000 iterations，32-byte key
指纹SHA256(derivedKey) → 64 字符 hex（小写比较）
对称加密AES-256-GCM（nonce 12 字节，tag 16 字节）
PMEM 布局Magic(4, PMEM) + version(1, 0x01) + flag(1, 0x01 表示加密) + salt(32) + AES-GCM combined(12+cipher+16)
salt 生成SecRandomCopyBytes / crypto/rand，32 字节，Base64 存储
实现一致性Sources/Engine/Sync/SyncCrypto.swift / Sources/Engine/DataPorterCrypto.swift / sync-server/internal/pmemcrypto/ / web/index.html（PMEM_MAGIC+PBKDF2_ITER = 600000）

6.2 applyEncryptionChange

文件：Sources/Engine/Sync/SyncEncryption.swift — applyEncryptionChange(enabled:passphrase:clientID:baseURL:token:)

enabled? ├─ NO → configChanged = wasEnabled └─ YES → wasEnabled & 已有 salt/fp 且口令验签通过? ├─ YES → setOwnPassphrase; return false (无重传) └─ NO → newSalt = random(32); newFP = keyFingerprint(pass, salt) configChanged = true configChanged? ├─ NO → return false └─ YES → DELETE /api/v1/clients/{id}/items (purgeServerItems, 120s 超时) → resetSyncWatermarks() (清 lastCompletedAt / peerLastPullAt / peerLastTrashSyncAt) → 写 UD: enabled / salt / fingerprint / passphrase → return true → UI 触发 syncNow()，performSync 自然走 since=nil 全量重传

注意：SyncError.passphraseRequired(clientID) 在拉取加密对端但本机无对应 peerPassphrase 时抛出，由 eligiblePeerIDs 过滤掉缺少口令的对端以避免冲突。

6.3 上传加密形态

encryptPayloadForUpload 生成内层 inner: SyncClipPayload（含完整 origin_*、metadata、4 个二进制 Base64、created_at、last_used_at），JSONEncoder.encode(inner) 后用 SyncCrypto.encryptPayloadJSON（内部走 DataPorterCrypto.encrypt）输出 PMEM Base64。

外层返回的 SyncClipPayload 仅保留：item_id / created_at / last_used_at / content_type / encrypted=true / payload_encrypted=...；其它字段（含 content、source_app、image_data_base64、origin_* 等）一律置 nil/false/""。

服务端 store.IngestSync 收到 encrypted=true 后只写 content_type、created_at、last_used_at、is_encrypted=1，把 PMEM Base64 落 item_attachments.payload_json（不写 items.content）。

6.4 Web 解密

index.html 中 decryptPayloadBase64 走 crypto.subtle.decrypt（仅在 HTTPS 或 localhost 安全上下文可用，否则显示警告）；指纹验证用 keyFingerprint(passphrase, salt)。图片通过 GET /api/v1/items/{c}/{i}/image 由服务端按格式检测后下发（JPEG/PNG/GIF 原样，否则在 macOS 上经 sips 转码为 JPEG；HEIC/TIFF 等不被 Chrome/Firefox 直接渲染）。

7. sync-server API

所有 JSON 响应在客户端声明 Accept-Encoding: gzip 时经 compressMiddleware gzip（BestSpeed）。Bearer 鉴权 4 个写接口；其余只读接口对外暴露，公网部署需网络层防护。

DELETE/api/v1/clients/{clientID}/trash—handlePurgeClientTrash永久删除该 Client 全部回收站项，返回 {deleted_count}
GET/favicon.ico—handleFavicon站点图标，Cache-Control 1 天
GET/—handleDashboardHTML Dashboard（text/template 渲染 web/index.html）

分页：默认 limit 30（defaultItemPageSize），最大 100（maxItemPageSize）；游标 ItemCursor = (created_at, item_id[, deleted_at])，next_cursor 由服务端基于 limit+1 判断 has_more 后回填。

排序：items / pull 用 created_at DESC, item_id DESC；trash 用 deleted_at DESC, item_id DESC（次级键 item_id 保证同时间戳分页稳定）。

Ingest 时 origin 默认值（store.IngestSync）：origin_client_id ← item 或 req.ClientID；origin_hostname ← item 或 req.Hostname；origin_ip ← item 或 clientIP(r, trustProxy)（trust_proxy=true 时优先取 X-Forwarded-For/X-Real-IP）。

8. Web 仪表盘

单页应用：sync-server/internal/httpserver/web/index.html，构建时 //go:embed。

8.1 主要状态

state = {
  clientId, type, trash, cursor, loading, hasMore,
  clientEncryption, browsePassphrase
}

8.2 功能模块

  • Browse — URL 参数 ?client=&type=&trash=1；history.pushState 同步 URL；IntersectionObserver 无限滚动加载。
  • 加密门控 — fingerprint 校验；sessionStorage 双 key：
    • sync-pass:{clientId} — 保存口令原文
    • sync-pass-verified:{clientId} — "1" 表示已通过指纹校验
    二次进入若仅有 sync-pass 但 verified 缺失，会再次跑 verifyBrowsePassphrase。
  • 渲染 — 列表卡片 / image 瀑布流（响应式 5→1 列）；加密项浏览器内 crypto.subtle.decrypt 后显示缩略图与详情（applyDecryptedItemUI）。
  • 回收站 — Trash 切换 → 重新加载；Empty recycle bin → DELETE /api/v1/clients/{id}/trash。
  • 主题 — localStorage['pastememo-sync-theme'] 持久化 light/dark；HTML 根 data-theme 属性驱动。

9. 水印与状态

lastPullAt[peer]syncPeerLastPullAtByClientpullFromPeer 整轮结束（中间分页失败则不更新，since 保留为上一成功值）GET /pull?since=
lastTrashSyncAt[id]syncPeerLastTrashSyncAtByClientsyncTrashFromClient 整轮结束（同上失败保护）GET /trash?since=

SyncEncryption.resetSyncWatermarks() 清除上述三者 — 加密配置变更（开启/关闭/换口令）时调用，触发 performSync 走 since = nil 的全量重传。

失败时 syncNow 仅写 lastError 与 autoPaused = true，不推进水印，下一次 syncNow 会从上次成功点继续。

10. 错误处理

10.1 SyncError

Case触发
notConfiguredURL / token 任一为空
invalidURLendpoint 构建失败（URLComponents.url 为 nil 等）
noItemsuploaded == 0 && downloaded == 0 && deleted == 0；Scheduler 视为成功
httpStatus(code, body)响应非 2xx
serverMessage(String)JSON decode 失败、非 HTTPResponse、响应缺失 payload_encrypted 等
transport(Error)URLSession 抛出（连接被拒、超时等）
passphraseRequired(peerID)上传需本机口令、加密对端缺 peerPassphrase、加密 payload 缺口令

CryptoError（keyDerivationFailed、invalidFormat、wrongPassword）由 SyncCrypto 与 DataPorterCrypto 抛出，被 SyncEngine 包装为 SyncError.serverMessage。
