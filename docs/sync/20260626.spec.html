1. 概述

PasteMemo 同步功能允许用户在多台 Mac 之间，通过自托管的 sync-server 共享剪贴板历史。 客户端以增量方式上传本机产生的条目，并可选择从其它设备拉取条目；服务端提供 Web 仪表盘浏览与管理数据。

2. 目标与范围

2.1 目标

  • 在多台 Mac 间共享剪贴板历史，无需第三方云剪贴板服务。
  • 用户完全掌控数据：自托管服务器、可选端到端加密。
  • 增量同步，降低带宽与服务器负载。
  • 记录条目来源设备信息，便于区分本机与其它设备内容。

2.2 范围内

  • macOS 客户端设置页「同步」模块。
  • Go 语言 sync-server（API + Web UI）。
  • 上传、对端拉取、回收站删除同步、加密、来源元数据。

2.3 范围外

  • 本机删除剪贴板条目同步到服务器（单向：服务器 → 客户端）。
  • 已上传条目的内容修改/版本更新同步。
  • iOS / Windows 客户端。
  • 账户体系、多租户、细粒度 ACL。

3. 术语

术语说明
Client ID每台 PasteMemo 实例的唯一 KSUID（27 字符 base62，时间排序），由 KSUID.generate() 生成，标识同步身份。
KSUID27 字符 base62 编码的时间排序唯一 ID（segmentio/ksuid 兼容），用作 ClipItem.itemID 的跨设备主键。生成见 Sources/Engine/KSUID.swift。
Sync Token客户端与服务器共享的 Bearer 密钥，用于 API 鉴权。
对端 (Peer)同一 sync-server 上其它 Client ID 对应的设备。
水印 (Watermark)记录上次成功同步时间点，用于增量上传/拉取。
本机条目originClientID 为空或等于本机 Client ID 的条目。
对端条目从其它设备拉取、originClientID 为对端 ID 的条目。
PMEMPasteMemo 导出/加密使用的容器格式（Magic + PBKDF2 + AES-GCM）。
Item Cursor分页定位符：(created_at, item_id) 或 (deleted_at, item_id)，保证同时间戳下分页稳定。

4. 用户角色

角色描述主要操作
终端用户使用 PasteMemo 的 Mac 用户配置同步、手动/自动同步、选择对端、设置加密口令
服务器管理员部署 sync-server 的运维人员配置 Token、数据库路径、反向代理、备份 SQLite
浏览者通过浏览器访问 Web 仪表盘浏览条目、删除/恢复、清空回收站、解密查看（若加密）

5. 功能需求

5.1 服务器与认证

REQ-SRV-001
必须 服务器通过环境变量 SYNC_TOKEN 配置共享密钥；未配置则拒绝启动。
REQ-SRV-002
必须 上传 (POST /api/v1/sync)、拉取 (GET /api/v1/pull)、回收站列表 (GET /api/v1/trash)、清空客户端数据 (DELETE /api/v1/clients/{id}/items) 须携带 Authorization: Bearer <token>。
REQ-SRV-003
应当 生产环境通过 HTTPS 与反向代理部署；支持 SYNC_TRUST_PROXY 获取真实客户端 IP。

5.2 客户端配置

REQ-CLI-001
必须 用户可配置：服务器 URL、Token、是否启用自动同步、同步间隔（分钟）、批次大小。
REQ-CLI-002
必须 首次使用时自动生成 Client ID（27 字符 base62 KSUID），支持复制与重新生成。重新生成不会迁移服务器上旧 ID 的数据。
REQ-CLI-003
必须 默认服务器 URL 为 http://127.0.0.1:8787；默认同步间隔 5 分钟；默认批次 20 条。
REQ-CLI-004
必须 同步间隔范围 1–10080 分钟（7 天）；批次大小范围 1–500。
REQ-CLI-005
应当 单二进制字段上限 10 MiB（超出则清空该字段并标记 truncated）；单请求体上限 48 MiB；超限由 uploadBatchAdaptive 自动二分拆分 batch，单条仍超限时再次 stripBinaryFields。

5.3 上传同步

REQ-UPL-001
必须 客户端将 createdAt 晚于上次成功同步时间的本机条目分批上传至服务器。
REQ-UPL-002
必须 从其它设备拉取的条目（对端条目）不得再次上传，包括全量重同步场景。
REQ-UPL-003
必须 服务器按 (client_id, item_id) 去重；重复条目计入 deduped_count，不报错。
REQ-UPL-004
必须 上传请求携带本机 hostname；每条目携带来源 Client ID、Hostname、IP（见 5.8）。
REQ-UPL-005
应当 单字段二进制上限 10 MiB；超出则省略该字段并标记 truncated。

5.4 对端拉取

REQ-PULL-001
必须 用户可从服务器发现的设备列表中选择对端；拉取对端自上次拉取以来新增的条目。
REQ-PULL-002
必须 本地已存在相同 item_id 的条目时跳过导入（幂等）。
REQ-PULL-003
必须 对端启用加密时，须先输入并验证对端口令后方可选择该对端；口令指纹与服务器公布的 salt/fingerprint 一致。
REQ-PULL-004
必须 导入时写入来源 Client ID、Hostname、IP；属性面板可查看（见 5.8）。

5.5 回收站同步

REQ-TRASH-001
必须 Web 仪表盘删除条目为软删除（进入回收站）；客户端通过对端/本机 trash API 拉取删除记录并本地删除对应 item_id。
REQ-TRASH-002
必须 回收站条目保留 10 天后由服务器自动永久清除。
REQ-TRASH-003
必须 Web 回收站视图提供「全部清除」，永久删除该 Client 下所有回收站条目。
REQ-TRASH-005
应当 回收站/拉取接口基于 created_at DESC, item_id DESC（或 deleted_at DESC, item_id DESC）的稳定排序返回，由 cursor_created_at+cursor_item_id（或 cursor_deleted_at+cursor_item_id）定位下一页。
REQ-TRASH-004
必须 客户端在 PasteMemo 内删除条目不会同步到服务器。

5.6 定时同步

REQ-SCH-001
必须 启用自动同步后，按配置间隔执行完整同步流程（上传 → 拉取 → 回收站）。
REQ-SCH-002
必须 提供「立即同步」手动触发；同步进行中禁止重复触发。
REQ-SCH-003
必须 同步失败时自动暂停定时同步（autoPaused=true、stop() 定时器、记录 lastError），显示错误信息；用户可「恢复自动同步」清 autoPaused 并重新 reschedule()。
REQ-SCH-004
应当 当本次无新上传、无新下载、无新删除时，视为成功（无新内容），仍推进上传水印。

5.7 端到端加密

REQ-ENC-001
必须 用户可启用上传加密；口令经 PBKDF2（600000 次）派生密钥，AES-GCM 加密条目 JSON（PMEM 格式）。
REQ-ENC-002
必须 服务器仅存储密文与密钥指纹、salt；不存储口令或明文密钥。
REQ-ENC-003
必须 启用/更改加密或口令时：删除服务器上该 Client 的全部条目、重置所有同步水印、自动触发仅含本机条目的全量重传。
REQ-ENC-004
必须 对端加密条目拉取时须持有对端口令；本机加密上传须持有本机口令。
REQ-ENC-005
应当 Web 仪表盘浏览加密 Client 时，用户输入口令后在浏览器内解密预览（Web Crypto）。

5.8 来源追踪

REQ-ORG-001
必须 本机新捕获的剪贴板条目记录：来源 Client ID（KSUID）、Hostname、IPv4。
REQ-ORG-002
必须 从对端导入的条目保留/补全来源三元组；快捷面板与管理器属性列表展示上述字段。
REQ-ORG-003
必须 来源信息随条目 payload 同步至服务器 origin_* 字段持久化。

5.9 Web 管理台

REQ-WEB-001
必须 首页展示已连接 Client 列表：ID、Hostname、IP、同步统计、条目数、是否加密。
REQ-WEB-002
必须 按 Client 浏览条目：类型筛选、分页滚动加载、列表/图片瀑布流视图。
REQ-WEB-003
必须 条目详情弹窗；支持删除（进回收站）、回收站内恢复。
REQ-WEB-004
应当 支持明暗主题切换（localStorage['pastememo-sync-theme']）；加密 Client 浏览前口令门控，浏览器侧 crypto.subtle.decrypt 解密 PMEM；口令存于 sessionStorage['sync-pass:{clientId}'] 与验证标志 sync-pass-verified:{clientId}。
REQ-WEB-005
可选 服务端 JSON 响应在客户端 Accept-Encoding: gzip 时启用 gzip（BestSpeed）；图片预览走 GET /api/v1/items/{c}/{i}/image，JPEG/PNG/GIF 原样下发，HEIC/TIFF 等在 macOS 上经 sips 转 JPEG；非 Darwin 不支持转码时返回 415。

6. 非功能需求

类别要求
性能默认批次 20 条；单请求体客户端上限 48 MiB（超限自适应拆分）；服务端上传上限 512 MiB。
可靠性同步失败不丢本地数据；dedup 保证重复上传安全。
可用性设置页同步模块支持窗口缩放与内容区滚动。
安全Token 鉴权保护写入类 API；生产环境 HTTPS；加密可选；仪表盘默认无鉴权（需网络层防护）。
可维护性SQLite 单文件备份；Go 1.22+ 构建 sync-server。
兼容性旧条目无 originClientID 视为本机条目；旧同步间隔键自动迁移为分钟。

8. 数据与约束

8.1 同步条目标识

item_id 为跨设备唯一键；本机捕获时由 KSUID.generate() 生成 27 字符 base62 字符串（时间排序），导入时沿用对端 ID。

8.2 本机 vs 对端判定

originClientID是否上传
null / 空（旧数据）是（视为本机）
等于本机 Client ID是
其它 Client ID否

8.3 加密变更后的重同步

加密设置变更 → 服务器 DELETE 该 client 全部 items → 重置 lastCompletedAt、各 peer pull/trash 水印 → 仅上传本机条目。

9. 验收标准

  • 两台 Mac 配置同一 Token，A 复制文本，B 手动同步后可看到 A 的条目。
  • B 拉取的条目在 B 的属性中显示 A 的 Client ID / Hostname / IP。
  • B 不会将 A 的条目再次上传至服务器（服务器 item 计数不重复增长）。
  • Web 删除 A 的条目后，B 同步后本地对应条目消失。
  • 启用加密后服务器存密文；Web 输入正确口令可解密查看；错误口令拒绝。
  • 更改加密口令后仅本机历史重传，不含对端拉取条目。
  • 同步失败显示错误并暂停；恢复后继续按间隔同步。
  • 回收站「全部清除」后该 Client 回收站为空。
  • 设置页同步内容超出窗口高度时可滚动。

10. 已知限制

  • 本地删除不同步到服务器；条目在服务器上仍存在直至 Web 删除或过期。
  • 不支持已同步条目的内容更新（仅新增 + 删除传播）。
  • Web 仪表盘及多数只读 API 无 Bearer 鉴权，公网暴露需额外防护。
  • 加密口令存储于 UserDefaults（明文），依赖 macOS 用户会话安全。
  • 重新生成 Client ID 不会迁移服务器上旧 ID 的数据。
  • HEIC 等格式在 Web 仪表盘非 Safari 浏览器中无法直接渲染；服务端在 GET /api/v1/items/{c}/{i}/image 通过 sips（Darwin）转码为 JPEG 后再下发。
  • 单批次请求体超过 48 MiB 时客户端会二分拆分；单条超限则清空四个二进制字段并标记 truncated=true，仍超限时上报错误。
  • 每个 ClipItem 在 SyncEngine.performSync 末尾若 uploaded == 0 && downloaded == 0 && deleted == 0 会抛出 SyncError.noItems，由 SyncScheduler 视为成功（推进上传水印）。
安全提示：生产环境务必使用 HTTPS、强随机 Token，并通过防火墙或反向代理限制 Web 仪表盘访问。
