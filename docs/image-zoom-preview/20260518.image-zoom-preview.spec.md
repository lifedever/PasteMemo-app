# 图片缩放预览 — 需求说明（spec.md）

> 技术实现见 [tech.md](./tech.md)

## 背景

管理器窗口与快捷粘贴窗口中的图片预览原先统一使用「整图适应容器」（aspect fit）。当剪贴内容为超长截图、高分辨率大图时，预览区高度有限，整图被压缩后细节不可辨认。

现有补救方式依赖系统 Quick Look（空格）或「预览.app」，步骤额外；且 **Finder 复制的单文件图片** 在详情中曾仅显示文件图标。macOS **⌘⇧截图** 等写入剪贴板的图片常为 **TIFF/HEIC**，若用错误扩展名写入临时文件，「预览.app」会打不开（已在实现中修复）。

## 目标

1. 在**管理器详情**与**快捷粘贴预览区**提供应用内可滚动、可缩放的图片预览。
2. 默认以**适应预览区宽度**展示，长图可纵向滚动阅读，无需先把整图缩到极小。
3. 覆盖三类图片来源：纯位图、链接内嵌图（含 data URI）、文件路径单图。
4. 保留系统 Quick Look / 预览.app 作为**查看原图清晰度**的出口，不替代现有快捷键。

## 用户场景

| 场景 | 期望 |
|------|------|
| 手机长截图（极高宽高比） | 默认宽度可读，向下滚动查看全文；应用内放大受预览分辨率限制，要看清文字用「在预览中打开」 |
| 4K 截图 / macOS 区域截图 | 应用内可浏览全貌；原图清晰度通过预览.app |
| Finder 复制单张 `.png` | 详情/快捷粘贴显示可缩放位图，而非仅文件名 |
| 复制 `data:image/...` 链接 | 与位图条目一致的缩放体验 |

## 功能需求

### 展示与布局

- 图片在预览区内**适应宽度**（图片逻辑宽度 = 预览区可用宽度；高度按比例，超出部分通过纵向滚动查看）。
- 窄图允许放大至适应宽度（可大于 100% 逻辑缩放），宽图初始缩放 ≤ 100%。
- 加载过程：先显示缩略图占位，再替换为降采样预览图（`ImageCache`，与原先 `AsyncPreviewImageView` 相同策略）。

### 应用内清晰度说明（重要）

| 层级 | 说明 |
|------|------|
| **入库** | 纯剪贴板图（`content == "[Image]"`）的 `imageData` 为粘贴板**原始字节**，捕获时不缩小 |
| **应用内预览** | 为性能将最长边降采样（管理器约 2800px、快捷粘贴约 2200px 量级，见 tech.md） |
| **工具条「100%」** | 指**当前预览位图**的逻辑 1:1，不是显示器像素级原图 |
| **看清原图** | 使用「在预览中打开」、空格 Quick Look、或快捷粘贴双击（见下） |

超长图在降采样时按「最长边」封顶，短边可能很窄，再「适应宽度」放大时应用内会显得发糊——属预览策略限制，非入库丢失。

### 缩放与平移

- **捏合**（触控板双指）缩放。
- **⌘ + 滚轮**缩放（`NSScrollView` 原生放大）。
- 缩放范围：**10% – 800%**。
- 放大后可通过滚动条/拖拽查看溢出区域。

### 快捷操作与入口

预览区**右下角**半透明胶囊工具条（**仅图标、无文字**；图片加载完成后才显示）：

| 图标 | 功能 | 悬停提示 |
|------|------|----------|
| 四角箭头 | 适应宽度 | `preview.image.zoomFit` |
| `1` + 放大镜 | 100%（预览分辨率） | `preview.image.zoomActual` |
| 眼睛 | **在预览中打开**（预览.app） | `preview.image.openInPreview` |

**不是**菜单项；若看不到工具条，请确认已运行含本功能的构建且预览图已加载完成。

其它入口：

| 入口 | 管理器 | 快捷粘贴 |
|------|--------|----------|
| 双击预览图 | 适应宽度 ↔ 100% | 用**预览.app**打开原图 |
| 空格 | Quick Look 浮层 | — |
| ⌘O | — | Quick Look 浮层（非预览.app） |
| ⌘K →「用“预览”打开」 | 有（子快捷键 L） | 有（子快捷键 L） |

### 在预览中打开 — 适用范围

| 类型 | 条件 | 行为 |
|------|------|------|
| A 纯位图 | `contentType == .image`，有 `imageBytesForExport()` 或 `imageData` | 按真实格式写临时文件（png/jpg/tiff/heic…） |
| B 链接图 | `imageData` 或可归解码的 `data:image/...` | 同上 |
| C 文件路径图 | 磁盘原文件存在 | 直接打开路径；不存在则回退内存字节 |
| 普通 http(s) 链接 | 无内嵌图数据 | **不**提供「在预览中打开」（仍可用 Web 预览） |

macOS 截图（⌘⇧3/4/5 进剪贴板）走类型 A，必须使用正确扩展名临时文件，否则预览.app 无法打开（回归已修）。

### 双击行为

- **管理器详情**：在「适应宽度」与「100%」之间切换。
- **快捷粘贴**：双击打开**预览.app**（与工具条眼睛图标相同）；缩放用手势或工具条前两项。

### 适用范围（缩放预览）

| 类型 | 条件 |
|------|------|
| A 纯位图 | `contentType == .image` 且 `imageData != nil` 且 `content == "[Image]"` |
| B 链接图 | `contentType == .link` 且（`imageData != nil` 或 `data:image/...` 可解码） |
| C 文件路径图 | `contentType == .image` 且单一路径且可读原图/缩略图；多路径仍用文件列表 |

### 其它窗口行为

- 快捷粘贴：OCR 卡片仍在图片下方；**Esc** 仍优先关闭 Quick Look 面板，再关闭快捷粘贴面板。
- 敏感条目：仍由 `SensitiveMask` 包裹，遮罩内为可缩放预览。
- 列表行 36×36 缩略图**不**改动。

## 边界与异常

- 原图文件已删除：回退至 `imageData` / `imageBytesForExport()`；若无数据则占位，且「在预览中打开」不可用。
- 超大文件：沿用 `loadOriginalImageData` 体积上限；应用内预览仍降采样。
- 离线模式 / 关闭图片链接预览：data URI 与链接图行为与现有设置一致。
- 命令面板「用“预览”打开」：`canOpenInPreview` 与 `prepareURL` 一致，仅在实际能生成可打开 URL 时显示。

## 验收标准

- [ ] 管理器：超长截图默认宽度可读，可纵向滚动全文。
- [ ] 管理器：⌘+滚轮/捏合有效；工具条三项可见且可点（含眼睛）。
- [ ] 管理器：⌘⇧截图条目可用「在预览中打开」/ 命令面板打开预览.app。
- [ ] 管理器：Finder 单文件图片为位图缩放预览（非仅图标）。
- [ ] 快捷粘贴：同上；OCR 在图下；双击打开预览.app。
- [ ] 快捷粘贴：`data:image` 可缩放且可「在预览中打开」。
- [ ] 原图路径失效：不崩溃；有缩略图回退；无法打开时不显示无效命令。
- [ ] 快速切换条目：无旧图残留。
- [ ] 敏感条目：遮罩内可缩放。

## 非目标

- 列表缩略图缩放。
- 应用内无限清晰度放大（不加载全分辨率进 `NSScrollView`）。
- 图片编辑、标注、裁剪。
- 替代 Quick Look / 移除空格预览。
- Relay 窗口预览。
- 多文件图片路径的网格预览。

## 已知限制与后续可选

- 应用内放大长图细节有限，依赖预览.app 看原图。
- 可选后续：长图按「最小宽度」降采样、或捏合时按需加载更高分辨率档位。
